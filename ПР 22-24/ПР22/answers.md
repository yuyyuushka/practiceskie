1. Преимущества миграций
Контроль версий схемы БД
Автоматизация применений изменений
Совместная работа в команде
Воспроизводимость развертываний

2. Типы связей между таблицами
Один-ко-многим (самый распространенный)
Многие-ко-многим (через промежуточную таблицу)
Один-к-одному (редко, для разделения таблиц)

3. ORM и преимущества
ORM — объектно-реляционное отображение, технология для работы с БД через объекты кода.
Преимущества: повышение производительности разработки, безопасность от SQL-инъекций, абстракция от конкретной СУБД.

4. Пагинация на уровне БД
Использование операторов LIMIT и OFFSET (или аналогов в других СУБД) для получения конкретной порции записей.

5. Транзакции
Группа последовательных операций с БД, которая выполняется как одно целое. Используются для обеспечения целостности данных, когда нужно либо выполнить все операции блока, либо ни одной.

6. Методы оптимизации запросов
Создание индексов
Нормализация и денормализация таблиц
Оптимизация структуры запросов (избегать N+1 проблемы)
Кэширование
Использование EXPLAIN для анализа

7. Обработка ошибок
Использование try-catch блоков
Проверка кодов возврата и исключений
Логирование ошибок с контекстом
Грейсфул-деградейшн (продолжение работы при ошибках БД, если возможно)

8. Разница между соединениями таблиц
Внутреннее соединение: возвращает только те записи, для которых есть соответствие в обеих таблицах.
Внешнее соединение: возвращает все записи из одной таблицы и соответствующие им записи из другой. Если соответствий нет, возвращает NULL.
