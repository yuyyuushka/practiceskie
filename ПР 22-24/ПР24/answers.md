1. Какие методы HTTP используются для CRUD операций и в каких случаях?
GET - получение данных
POST - создание новых данных
PUT/PATCH - полное/частичное обновление
DELETE - удаление данных

2. Как организовать обработку ошибок на фронтенде при работе с API?
Интерцепторы axios для централизованной обработки
Try/catch в компонентах
Глобальный error boundary в React
Пользовательские уведомления об ошибках

3. Что такое CORS и как его настроить для разработки?
Механизм безопасности браузеров для кросс-доменных запросов
Настройка: app.use(cors()) в Express
Для продакшена указать конкретные домены

4. Какие подходы к управлению состоянием при интеграции с API вы знаете?
Локальное состояние (useState)
Глобальное состояние (Redux, Context API)
Серверное состояние (React Query, SWR)
Кастомные хуки для инкапсуляции логики

5. Как реализовать индикацию загрузки без ухудшения UX?
Оптимистичные обновления
Скелетон-загрузчики вместо спиннеров
Пороговые значения (задержка показа индикатора)
Прогресс-бары для длительных операций

6. В чем преимущества и недостатки оптимистичных обновлений?
Плюсы: мгновенный отклик, улучшенный UX
Минусы: сложность реализации, риск рассинхронизации, необходимость отката

7. Как организовать валидацию данных на клиенте и сервере?
Клиент: формальная валидация для UX
Сервер: бизнес-логика и безопасность
Единая схема валидации (например, Zod)
Детальные сообщения об ошибках

8. Какие методы кэширования данных с сервера можно использовать?
HTTP-кэширование (ETag, Cache-Control)
In-memory кэш (Redis)
Локальное кэширование (localStorage)
Библиотеки (React Query, SWR) с автоматической инвалидацией
