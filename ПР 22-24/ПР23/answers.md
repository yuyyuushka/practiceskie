
1.	В чем разница между аутентификацией и авторизацией?
Аутентификация - проверка "кто ты?" (верификация личности)
Авторизация - проверка "что тебе разрешено?" (права доступа)

2.	Какие преимущества и недостатки у JWT по сравнению с сессиями?
Преимущества JWT:
Stateless (не требует хранения на сервере)
Масштабируемость
Кросс-доменная работа

Недостатки JWT:
Невозможность отозвать токен до истечения срока
Больший размер данных
Сложность управления refresh tokens

3.	Почему важно хешировать пароли и как работает bcrypt?
Зачем: защита при утечке БД
Как работает: медленный алгоритм + salt → устойчивость к brute-force

4.	Что такое salt в контексте хеширования паролей?
Случайные данные, добавляемые к паролю перед хешированием
Предотвращает атаки по радужным таблицам
Уникальна для каждого пароля

5.	Как организовать обновление JWT токенов (refresh tokens)?
// Механизм:
Access Token (15 мин) + Refresh Token (7 дней)
// При истечении access token:
POST /api/auth/refresh { refreshToken }
Новый access token

6.	Какие методы защиты от атак типа brute-force вы знаете?
Rate limiting (ограничение попыток)
CAPTCHA после нескольких неудачных попыток
Блокировка IP на время
Увеличение задержки ответа

7.	Как реализовать систему ролей и разрешений?
// Иерархия:
Пользователь Роли Разрешения
// Пример:
User ['read_own_data']
Admin ['read_any_data', 'write_any_data']

8.	Какие меры безопасности важно соблюдать при работе с JWT?
Использовать HTTPS
Короткое время жизни access token
Secure хранение на клиенте (httpOnly cookies)
Проверка подписи токена
Не хранить чувствительные данные в payload
