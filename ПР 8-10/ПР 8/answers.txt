1.	Какие инструменты графического редактора наиболее полезны для извлечения информации о размерах, цветах и шрифтах?
Линейка, пипетка, текстовый инструмент, инструмент автоматического определения отступов и параметров.

2.	Как правильно организовать структуру CSS-переменных на основе данных из макета?
Группировать переменные по логическим блокам (цвета, шрифты, отступы, размеры экранов), использовать осмысленные имена.

3.	Какие метрики из макета наиболее критичны для точной верстки (размеры, отступы, шрифты)?
Межстрочный интервал (line-height), межбуквенный интервал (letter-spacing), размер шрифта, все отступы (margin/padding), высота строки.

4.	Как определить систему сетки, используемую в макете и перенести ее в CSS Grid или Flexbox?
Анализировать расположение и выравнивание элементов для выбора между Grid (сложные двумерные layout) и Flexbox (однонаправленные компоненты). Определять промежутки и колонки.

5.	Какие сложности возникают при переносе типографики из графического макета в веб-страницу?
Несовпадение механизмов рендеринга шрифтов, различия в метриках строк и межбуквенных интервалов, ограниченность доступных свойств CSS.

6.	Как обрабатывать адаптивность, когда в макете представлен только десктопный вариант?
Использовать мобильный-first подход, логически экстраполировать изменения layout, опираться на типографическую шкалу и относительные единицы (%, rem, vw).

7.	Какие инструменты браузера помогают сравнивать сверстанную страницу с оригинальным макетом?
Инструменты разработчика (DevTools) для наложения скриншота макета, проверки размеров, шрифтов и цветов.

8.	Как правильно экспортировать изображения из графического редактора для веб-использования?
Выбирать правильный формат (SVG для графики, WebP/AVIF для фото, PNG для прозрачности, JPG для сложных изображений без прозрачности), оптимизировать вес, экспортировать в 2x размере для ретины.

9.	Что делать, если в макете используются шрифты, которых нет в веб-доступе?
Найти и подключить веб-безопасный аналог, использовать сервисы для веб-шрифтов (Google Fonts) или кастомизировать подключение через @font-face.

10.	Какие методологии помогают поддерживать соответствие между макетом и кодом при дальнейших изменениях?
Методологии именования CSS (БЭМ), компонентный подход, использование CSS-препроцессоров, строгая организация файловой структуры проекта, регрессионное тестирование.
