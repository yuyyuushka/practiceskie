1. В чем разница между управляемыми и неуправляемыми компонентами в формах React?
Управляемые компоненты: значение поля управляется React через state, изменения обрабатываются через обработчики событий
Неуправляемые компоненты: значения полей хранятся в DOM, доступ к ним осуществляется через ref

2. Когда следует использовать управляемые компоненты, а когда - неуправляемые?
Управляемые: когда нужен мгновенный доступ к данным, валидация в реальном времени, сложная логика обработки
Неуправляемые: когда форма простая, нужна высокая производительность, интеграция с не-React кодом

3. Какие методы используются для обработки событий в формах React?
onChange - для обработки изменений полей ввода
onSubmit - для обработки отправки формы
onBlur - для обработки потери фокуса
onFocus - для обработки получения фокуса

4. Как обрабатывать несколько полей ввода с одним обработчиком?
Использовать атрибут name у полей ввода
В обработчике использовать event.target.name для определения поля
Обновлять state динамически через computed property names

5. Какие подходы к валидации форм в React вы знаете?
Валидация на стороне клиента с помощью state и условий
Использование библиотек (Formik, React Hook Form, Yup)
HTML5 валидация через атрибуты required, pattern
Кастомные валидационные функции

6. Как предотвратить стандартное поведение формы (перезагрузку страницы)?
Вызвать event.preventDefault() в обработчике onSubmit

7. Что такое "поднятие состояния" (lifting state up) в контексте форм?
Это перенос состояния из дочерних компонентов в родительский компонент
Позволяет обмениваться данными между несколькими компонентами
Упрощает управление состоянием сложных форм
