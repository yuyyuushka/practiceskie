1. В чем разница между Link и NavLink в React Router?
`Link` используется для создания обычных ссылок между страницами
`NavLink` - это специальная версия `Link`, которая добавляет класс активного состояния, когда ссылка активна (совпадает с текущим URL)

2. Как получить параметры из URL в компоненте?
Используется хук `useParams()` для получения параметров маршрута
Например: `const { id } = useParams();` для пути `/products/:id`

3. Что такое программная навигация и как ее реализовать?
Программная навигация - это переход между страницами через код, а не через клик по ссылке
Реализуется с помощью хука `useNavigate()`: `const navigate = useNavigate(); navigate('/path');`

4. Как передать данные между маршрутами?
Через `state` в навигации: `navigate('/path', { state: { data: value } })`
Через параметры URL: `/path/:param`
Через query-параметры: `/path?key=value`

5. Что делает компонент Routes?
`Routes` заменяет старый `Switch` и используется для группировки маршрутов
Он рендерит только первый маршрут, который совпадает с текущим URL

7. В чем разница между useNavigate и redirect?
`useNavigate` - хук для программной навигации в функциональных компонентах
`redirect` - компонент для перенаправления, обычно используется в обработчиках загрузки данных или при рендеринге
