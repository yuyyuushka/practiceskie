1.	Какие HTML5-атрибуты используются для клиентской валидации и какую функциональность они предоставляют?
required, type="email/url/number", pattern, minlength/maxlength, min/max. Обеспечивают базовую проверку без JavaScript.

2.	В чем разница между novalidate атрибутом формы и отключением валидации через JavaScript?
novalidate отключает встроенную валидацию, но сохраняет возможность кастомной проверки. Отключение через JS полностью убирает валидацию.

3.	Как правильно связать <label> с полем ввода и почему это важно для доступности?
Атрибут for в <label> связывается с id поля. Важно для скринридеров и увеличения области клика.

4.	Какие типы полей input наиболее подходят для разных видов данных (email, телефон, пароль и т.д.)?
email - email с валидацией
tel - телефон
password - скрытый ввод
number - числовой ввод
date - выбор даты

5.	Как работает валидация с помощью pattern атрибута и в каких случаях его следует использовать?
Проверяет значение по регулярному выражению. Используется для кастомных форматов (почтовый индекс, номер карты).

6.	Какие события JavaScript используются для реализации валидации в реальном времени и почему?
input (реальное время), change (после изменения), blur (при потере фокуса). input предпочтительнее для мгновенной обратной связи.

7.	Как обеспечить доступность сообщений об ошибках для пользователей скринридеров?
ARIA-атрибуты: aria-invalid, aria-describedby для связи с сообщениями об ошибках. Динамическое обновление live-регионов.

8.	В чем преимущества использования fieldset и legend для группировки полей формы?
Группируют логически связанные поля. legend описывает группу. Улучшает навигацию для скринридеров и структуру формы.

9.	Как реализовать кастомную валидацию для сложных случаев (например, подтверждение пароля)?
Метод setCustomValidity() с проверкой в событиях. Для подтверждения пароля - сравнение значений двух полей.

10.	Какие методы CSS используются для визуального отображения состояний валидности/невалидности полей?
Псевдоклассы :valid, :invalid, :required, :optional. Стилизация через атрибут [aria-invalid="true"] для кастомного контроля.
