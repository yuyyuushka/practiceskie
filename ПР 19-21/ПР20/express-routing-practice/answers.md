1. В чем преимущества использования Express.js по сравнению с чистым Node.js для создания серверов?
Express.js предоставляет:
Упрощенный синтаксис для маршрутизации
Встроенные middleware функции
Автоматическую обработку заголовков и кодировок
Структурированную организацию кода
Большое количество готовых middleware
Более чистый и читаемый код

2. Что такое middleware в Express.js и каков порядок их выполнения?
Middleware - это функции, которые имеют доступ к объектам запроса (req), ответа (res) и следующей функции middleware в цикле "запрос-ответ". Они выполняются последовательно в порядке их объявления с помощью app.use() или app.METHOD().

3. Как организовать модульную структуру роутеров в Express приложении?
Модульная структура организуется через:
Создание отдельных файлов для каждого роутера
Использование express.Router()
Экспорт роутера через module.exports
Подключение в основном файле через app.use()

4. Какие методы роутинга поддерживает Express.js и для чего они используются?
Основные методы:
GET - получение данных
POST - создание новых данных
PUT - полное обновление данных
PATCH - частичное обновление данных
DELETE - удаление данных

5. Как обрабатывать параметры URL и query-параметры в Express?
Параметры URL через req.params (например /users/:id)
Query-параметры через req.query (например ?search=test)

6. В чем разница между app.use() и app.METHOD()?
app.use() - для middleware функций, которые выполняются для всех HTTP методов
app.METHOD() - для обработки конкретных HTTP методов (GET, POST, etc.)

7. Как организовать централизованную обработку ошибок в Express приложении?
Через специальный middleware с четырьмя параметрами:
app.use((err, req, res, next) => {
});
