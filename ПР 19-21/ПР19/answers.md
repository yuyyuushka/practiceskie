1. Что такое Node.js и в чем его преимущества для создания серверов?

Node.js - это среда выполнения JavaScript на стороне сервера, построенная на движке V8. Преимущества:
Однопоточная event-driven архитектура
Неблокирующий I/O
Высокая производительность
Единый язык для фронтенда и бэкенда
Большая экосистема npm

2. Как создать простой HTTP-сервер в Node.js?

Используя модуль `http`:
const http = require('http');
const server = http.createServer((req, res) => {
    res.end('Hello World');
});
server.listen(3000);


3. В чем разница между методами GET, POST, PUT и DELETE?
GET - получение данных
POST - создание новых данных
PUT - полное обновление существующих данных
DELETE - удаление данных

4. Что такое middleware и для чего оно используется?
Middleware - это функции, которые обрабатывают запросы перед основными обработчиками маршрутов. Используются для:
Логирования
Аутентификации
Парсинга тел запросов
Обслуживания статических файлов

5. Как обрабатывать различные HTTP-методы для одного URL?
Проверяя свойство req.method:

javascript
if (req.method === 'GET') {
    // обработка GET
} else if (req.method === 'POST') {
    // обработка POST
}

6. Что такое CORS и зачем нужны соответствующие заголовки?
CORS (Cross-Origin Resource Sharing) - механизм для безопасного межсайтового взаимодействия. Заголовки нужны для разрешения запросов с других доменов.

7. Как организовать чтение тела POST-запроса в Node.js?
С помощью событий data и end:

let body = '';
req.on('data', chunk => {
    body += chunk.toString();
});
req.on('end', () => {
    const data = JSON.parse(body);
});
